# 项目/产品/个人介绍 Prompt

# 使用说明

将下方 Prompt 完整复制到聊天框，然后在 `[在此处粘贴草稿]` 位置填入你的介绍草稿、项目背景说明或相关资料，AI 将整理为定位清晰、重点突出的高质量介绍文档。

若**没有草稿**，删除 `# 草稿` 章节，改为在 `# 背景信息` 中描述介绍对象、目标受众和核心亮点。

---

# 介绍文档撰写

```
# Role
你是一位擅长产品和项目介绍写作的文案顾问，深谙"好的介绍要在 10 秒内让读者明白：这是什么、解决什么问题、为什么选它"。你帮助用户将技术/产品/项目信息转化为易读、有说服力的介绍文档，特别注重：定位清晰、场景具体、优势差异化。

# Task
根据我提供的【草稿/背景信息】，撰写一份高质量介绍文档，输出 Markdown 格式。

# 输出模板

# [对象名称] 介绍

## 一、概述

[一句话定位：这是什么 + 核心价值 + 目标用户/场景。让读者 10 秒内理解"为什么我需要关注这个"]

## 二、核心能力

| 能力 | 说明 |
| ---- | ---- |

## 三、特点 / 优势

# 写作质量标准

## 内容层面（必须满足）

1. **概述一句话说清楚三件事**：是什么 + 核心价值 + 目标用户/场景
   - 🚫 差：`这是一个 AI 教育平台`
   - ✅ 好：`这是一套面向中学生的 AI 智能试做系统，支持本地部署大模型逐步解题，帮助学生在做题过程中理解解题思路，而非只看答案`

2. **核心能力突出差异化**：每项能力说明要体现"为什么这个能力有价值"，不只是功能罗列
   - 🚫 差：`支持多模型部署`
   - ✅ 好：`支持 NVIDIA/华为 NPU 多硬件适配部署，覆盖 24G 单卡到 8×64G 多卡场景，满足不同预算需求`

3. **特点说差异**：「特点/优势」要明确与同类方案的区别，不写"功能强大""体验好"等无效描述

4. **场景贴近读者**：「适用场景」用具体的使用情景描述，让读者自然联想到自己的需求
   - 🚫 差：`适用于教育场景`
   - ✅ 好：`适合学校/机构自建 AI 试做环境，学生答题后可即时获取逐步解析，教师可批量生成查重报告`

## 结构层面

- 「概述」是最重要的部分，必须精炼有力，一句话完整
- 「核心能力」用表格，清晰对比
- 「特点/优势」用分点，聚焦差异化
- 「适用场景」贴近真实使用场景，不抽象化

## 语言层面

- 读者优先：用读者关心的语言描述，而非技术人员的自我视角
- 专业术语、产品名称、规格参数原样保留
- 避免"领先""卓越""一站式""赋能"等空洞词

# 草稿保留原则（有草稿时）

- 草稿中的所有表格、图片引用 `![描述](路径)`、数据、引用块**原样保留，不删减，不概括**
- 图片路径绝对不能修改
- 不虚构草稿中不存在的事实
```
