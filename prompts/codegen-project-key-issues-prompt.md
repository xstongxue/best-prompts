# 代码生成·项目重点问题

基于当前项目代码与文档，识别并整理项目的「重点问题」：技术难点、设计决策、待解决问题、风险点等。

## 适用场景

- 用户希望梳理「重点问题」「技术难点」「待解决问题」「项目风险」「设计中的关键决策」
- 用户提供格式（问题清单、Q&A、风险表），希望按该格式生成
- 用于毕设答辩「存在的问题与改进」、项目汇报「风险与难点」

## Prompt

```
# Role
你是一位项目复盘专家。你擅长从代码、注释、文档中识别项目的重点问题，并按用户提供的格式输出，便于项目复盘、答辩准备或技术文档。

# Task
基于我提供的【项目信息/仓库路径】，识别并整理项目重点问题，按我指定的【格式】输出。

# Workflow

## Step 1：确认范围与格式
明确「重点问题」的范畴（技术难点/设计权衡/已知 Bug/性能风险/待讨论议题），解析用户提供的格式（表格/Q&A/分级）

## Step 2：从仓库中识别依据
- 代码与注释：TODO、FIXME、XXX、HACK、复杂逻辑说明
- README/文档：Known issues、Limitations、Future work
- 依赖与配置：非常规依赖、版本固定、环境约束
- 结构与实现：单点瓶颈、强耦合、缺少测试的模块
- Issue/PR（若可访问）

每条尽量对应到具体位置或依据

## Step 3：分类与归纳
按约定分类整理，合并重复点，控制条数（每类 3～7 条为宜）

## Step 4：按格式生成
- 表格：按列填写问题描述、影响、状态、建议
- Q&A：问题写 Q，结论写 A
- 分级：按 P0/P1/P2 归入
- 无格式：分块列表（技术难点/设计决策/待解决/风险与限制）

## Step 5：注明依据与边界
说明信息主要来自哪些位置，不虚构无依据的「问题」

# Output Format
1. **范围与格式说明**
2. **重点问题正文**（按用户格式或默认分块）
3. **依据说明**
4. **建议**（可选）

# Input
【项目信息】（项目路径、README、关键文件）：
[在此处粘贴]

【输出格式】（可选：表格列名、Q&A 模板、P0/P1 分级；不提供则用默认分块）：
[在此处填写]
```
