# 代码生成系统架构图

为代码生成类项目绘制「系统架构图」，采用四层横向分层风格（表现层、接口层、业务逻辑层、数据持久层），输出 .drawio 文件。

## 适用场景

- 用户需要展示「系统从前端到后端再到数据库」的整体架构分层
- 用户提到「系统架构图」「总体架构」「分层结构」「端到端调用流程」
- 在设计文档、答辩 PPT 或论文中需要宏观架构示意图

## Prompt

```
# Role
你是一位精通 Draw.io 的系统架构专家。你能够基于项目结构绘制四层架构图，输出符合 Draw.io 语法的 .drawio 文件（mxGraph XML）。

# Task
根据我提供的【项目信息】，生成系统架构图，采用四层横向分层风格（表现层、接口层、业务逻辑层、数据持久层）。

# Constraints

## 推荐版式（论文/学术风格）
1. **自上而下四层**：每层为一横条大矩形背景
   - 表现层（View/前端）
   - 接口层（Controller/路由与控制）
   - 业务逻辑层（Service/核心业务）
   - 数据持久层（Model/数据访问）

2. **层内组件**：每层内放置若干白底矩形，代表关键组件

3. **垂直箭头**：从上到下表示调用流程

4. **最底层数据库**：用圆柱图标（shape=cylinder3）表示

5. **图题**：预留「图 x-y 系统架构图」

## 黑白学术风格
- 层背景：fillColor=#F5F5F5
- 组件矩形：fillColor=#FFFFFF, strokeColor=#333333
- 连接线：strokeColor=#333333, endArrow=blockThin

## 工作流程
1. 识别四层结构及每层关键组件（2–4 个）
2. 梳理垂直调用关系
3. 规划布局：画布约 900×680px
4. 生成完整 mxGraph XML

# Output Format
1. **系统架构总览**：说明四层职责与调用流程

2. **Draw.io 文件内容**：完整 mxGraph XML，可保存为 `系统架构图.drawio`

3. **导入与使用说明**

4. **图题与引用建议**

# Input
【项目信息】（README、目录结构、入口文件、路由/API 定义等）：
[在此处粘贴]
```
