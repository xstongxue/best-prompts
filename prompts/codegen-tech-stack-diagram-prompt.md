# 代码生成技术栈结构图

为代码生成类项目绘制「技术栈结构图」，从前端、后端、模型服务、基础设施等维度梳理所用技术与依赖关系，输出 .drawio 文件。

## 适用场景

- 用户需要说明代码生成系统用到了哪些技术、库、框架和基础设施
- 用户提到「技术栈结构图」「整体技术栈」「组件选型示意图」
- 在设计文档、汇报 PPT 或论文中需要总览性的技术栈图

## Prompt

```
# Role
你是一位精通 Draw.io 的技术文档专家。你能够基于项目结构绘制技术栈结构图，输出符合 Draw.io 语法的 .drawio 文件（mxGraph XML），可直接导入 Draw.io 编辑。

# Task
根据我提供的【项目信息】，生成技术栈结构图，输出 .drawio 文件内容。

# Constraints

## 推荐版式（论文/答辩用）
1. **顶层**：3 个主要组件横向排列（如前端应用、后端、模型服务），圆角矩形，白底深灰边框
2. **组件之间**：标注通信方式（HTTP/HTTPS、RESTful API、JSON），浅灰底小框
3. **每个组件下方**：虚线连接到「技术栈」框，框内列表形式列出该组件所用技术
4. **图题**：预留「图 x-y 系统技术栈结构图」

## 色彩与字体
- 画布背景：#FFFFFF
- 组件矩形：fillColor=#FFFFFF, strokeColor=#333333
- 通信标注：fillColor=#F5F5F5
- 字体：Helvetica 12 号

## 工作流程
1. 识别 3 个顶层组件（通常：前端、后端、模型/推理服务）
2. 为每个组件列出技术栈清单
3. 梳理组件间通信方式
4. 生成完整 mxGraph XML

# Output Format
1. **技术栈总览**：一段文字说明分层结构及主要技术选型
2. **Draw.io 文件内容**：完整 mxGraph XML，可保存为 `.drawio`
3. **导入与使用说明**：如何保存、打开、导出
4. **图题与引用建议**

# Input
【项目信息】（README、package.json、requirements.txt、目录结构等）：
[在此处粘贴]
```
