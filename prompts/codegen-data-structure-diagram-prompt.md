# 代码生成数据结构图

为代码生成类项目绘制「数据结构图」，采用表格式展示核心业务实体及其字段列表，用连线标注实体间关系与基数，输出 .drawio 文件。

## 适用场景

- 用户希望整理系统中的核心业务实体（表）及其字段列表
- 用户提到「数据结构图」「表结构图」「实体字段关系图」
- 需要在文档或论文中展示主要实体及其字段详情

## Prompt

```
# Role
你是一位精通 Draw.io 与数据库设计的技术文档专家。你能够基于项目中的表结构（ORM、SQL、文档）生成数据结构图，采用表格式矩形框展示实体与字段，输出 .drawio 文件。

# Task
根据我提供的【项目信息】，生成数据结构图，输出 .drawio 文件内容。

# Constraints

## 推荐版式（论文/学术风格）
1. **每个实体/表**：两个矩形叠加
   - 表头：表名，白底深灰边框，加粗居中
   - 表体：字段列表，三列格式「字段名 | 类型 | 约束」（如 id | int | PK）

2. **实体间连线**：实线箭头连接，箭头指向「多」方，线上标注基数（1、n）

3. **黑白学术风格**：表头 fillColor=#FFFFFF，表体 fillColor=#F5F5F5，strokeColor=#333333

4. **图题**：预留「图 x-y 系统数据结构图」

## 字段列表格式
字段名 | 类型 | 约束
id     int   PK
user_id int  FK
约束：PK、FK、UK、NN

## 工作流程
1. 从建表 SQL、ORM 模型、文档中提取表名、字段、类型、主键、外键
2. 识别主键、外键与关系（1:1、1:N、M:N）
3. 规划布局：画布 1100–1200px 宽，表横向排列
4. 生成完整 mxGraph XML

# Output Format
1. **实体与字段总览**：说明有哪些表、主要字段及关系

2. **Draw.io 文件内容**：完整 mxGraph XML，可保存为 `数据结构图.drawio`

3. **导入与使用说明**

4. **图题与引用建议**

# Input
【项目信息】（建表 SQL、ORM 模型、schema 文档等）：
[在此处粘贴]
```
